mobilAP Documentation - Extending and Contributing to mobilAP

mobilAP is an open source project. Because of that, everything you might need to add on or enhance it is right in the source code. Because of the liberal license, you can feel free to alter it to suit your needs. The page will outline some of the ways you can both alter mobilAP or participate in its future development

Basic design

mobilAP was built using Apple's Dashcode framework. This is a HTML/CSS/Javascript toolkit that was designed to produce code for both standard desktop browsers as well as the MobileSafari browser present in the iPhone and iPod touch. The framework contains parts that contain behavior and looks similar to built in controls on the system. It is beyond the scope of this documentation to outline all the parts and their behavior, but more information can be found in the Dashcode Users Guide.

Folder Structure

As much as it is feasible, the files created by dashcode are separated from mobilAP specific files. Dashcode creates several folders used by the application

  • mobile - Code and images specific to the mobile (iPhone OS) product
  • Parts - Shared libraries and code
  • safari - Code and images specific to the desktop product

Inside the mobile and safari folders there are several files and folders:

  • Images - image assets used by that product. These images can be updated as desired
  • index.html - Base HTML. Since the majority of the content is loaded via AJAX, this is mostly a shell. The index.html for the safari product is located in the root
  • main.css - The primary style sheet. It has been grouped as much as feasible to ease in editing. It is recommended you use a style inspector such as the web inspected in Safari to trace style rules
  • main.js - The primary javascript file for this product. It contains the majority of the product specific code.
  • Parts - A folder with the dashcode parts definitions as well as parts Images. Since these are automatically generated by dashcode, take care in editing them

CSS and image alterations

The easiest alterations to make are style changes. If you wish to make changes, it is recommended that you use a CSS inspector such as Safari's web inspector. This will make finding the correct style easier. Edit the main.css file in both the mobile and safari folders as needed.

Contributing to mobilAP

If you have additional features, bug reports or other code alterations, you can begin examining the subversion repository found at http://svn.daap.uc.edu/repos/public/mobilAP/trunk
