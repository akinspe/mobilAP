mobilAP Documentation - Using Mac OS X

Since Mac OS X is a very common platform for mobilAP and is used by people who may not be web server administrators, this page has been created to assist those with setting up their Mac OS X environment.

Every Mac OS X system from 10.5 (Leopard) and above has everything you need to deploy mobilAP. The instructions to configure the service, however, are different based on whether you are using Mac OS X Client or Server.

  • Mac OS X Server
  • Mac OS X Client (i.e. not server)

Mac OS X Server (10.5 and 10.6)

Mac OS X Server includes many services ready for use. It is important to note, however, that this document does not address key aspects of web server administration including maintenance and security that are critical to running a public web server. Since Mac OS X server includes Apache, the world's most popular web server, there are many resources available that are applicable to this setup.

Enabling Apache and PHP on Mac OS X server is incredibly simple. You will need to use Server Admin, part of the Server Administration tools to connect to your server. If you have never used these tools, please consult the Mac OS X Server Documentation.

  1. Connect to your server using Server Admin. This can be done either on the server or remotely. You will need an administrative username and password.
  2. Click on your server in the Servers list on the left
  3. Click Settings in the toolbar
  4. Click Services in tab bar.
  5. Ensure that Web is selected and click save
  6. Click on the web service in the server list underneath your server
  7. Click Settings in toolbar
  8. Click Modules in the tab bar
  9. Ensure that the php5_module is checked and click save
  10. If it is not already started, click Start Web at the bottom of server admin to start the web service

If all is well you should be able to view your web server using its address. You can now continue with the rest of the installation instructions. You will typically want to place your files in the /Library/WebServer/Documents folder.

Mac OS X Client (10.5 and 10.6)

When we say "client" we are talking about the non server version of Mac OS X that comes with any Apple system. Your desktop, your laptop, etc. Even these systems include a version of Apache and PHP suitable for mobilAP use. You simply have to enable them. Note: Enabling PHP requires editing a system file. You will need to use a text editor capable of editing protected documents and an administrator name and password. One excellent free text editor that makes editing system files easy is TextWranger from BareBones software. Note that TextEdit does not currently include such a feature.

An alternative is to use a prebuilt package that automatically handles this configuration for you. Marc Liyanage provides such a package.

The following instructions assume you will be doing this yourself and not using any prebuilt packages. It also assumes that you have made no modifications to your existing Apache or PHP setup.

Enabling PHP

  1. Open /etc/apache2/httpd.conf in a text editor (since the /etc folder is normally hidden you can use the "Go to folder" shortcut (Shift-Cmd-G) and type in /etc/apache2 to see this folder)
  2. Use the find feature of your editor (typically Cmd-F) and find a line that contains LoadModule php5_module
  3. If present, remove the hash symbol (#, called a pound sign in other contexts) from the beginning of the line found in the previous step
  4. Save the file. You will likely need to type in an administrator name and password.

Starting Apache

  1. Open System Preferences
  2. Click on Sharing
  3. Click the box next to Web Sharing

If all is well you can open up http://localhost and view your site. You can now continue with the rest of the installation instructions. You will typically want to place your files in the /Library/WebServer/Documents folder. Note: You should turn off Apache by unchecking the checkbox in the Sharing panel of System Preferences if you are done testing. Unless you are planning on doing full time web development, keeping the web server running will use up system resources and could be a security risk as well.
