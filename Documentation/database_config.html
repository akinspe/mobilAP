mobilAP Documentation - Manual Database Configuration

Like most web applications, mobilAP uses a configuration file to configure its connection to its database. mobilAP supports both MySQL and SQLite for database storage. If you are unsure of which to choose, please read choosing a database for more information.

mobilAP attempts to use the setup routine to configure the database parameters. This requires write access to the mobilAP/data/mobilAP_dbconfig.php file. If this file cannot be written by the web server then you will need to configure it manually.

  1. Open up mobilAP/data/mobilAP_dbconfig.php in a text editor such as TextEdit.
  2. Find the line with db_type. Inside the single quote marks, enter either 'mysql' or 'sqlite', correctly done it should look similar to this:
    'db_type'=>'sqlite', // should be either 'mysql' or 'sqlite'
    Make sure that the comma is still present at the end of the line (the double slashes indicate a comment and can remain or be removed)
  3. If you are using SQLite then the only parameter you will use is db_folder. This folder must be writable by the webserver. You should enter the full path of the folder where you wish to place the SQLite database (in SQLite the entire database exists as a single file, however the server needs write access to the folder as well for temporary files). Note that the setup routine will attempt to create the database.
  4. If you are using MySQL then enter the host, user, password and database similarly in lines after db_type. It is not necessary to alter the db_folder parameter.

Take care to only enter in data between the quote marks. Do not add or remove any characters outside of this or you could render the site unusable.
