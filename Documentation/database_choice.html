mobilAP Documentation - Choosing a Database

mobilAP uses a database to store all the data used by the site. Version 2.0 supports using either SQLite or MySQL based databases. Each of these systems has its own strengths and weaknesses.

SQLite (recommended for most installations)

SQLite is a high performance embedded database system. Instead of using a dedicated server, SQLite stores its data in a single file and operations read this file directly. It is incredibly simple and fast and requires zero configuration or maintenance. The only requirement it has for mobilAP is that the web server must be able to write to the folder the database file is located. By default this is the mobilAP/data folder. You have the option to store this file in another location if you wish, as long as the web server can write to that folder. The default folder is located inside the mobilAP web tree. This would allow anyone, by default, to download the SQLite file and view it. If this is a concern, you should place the file in an alternate location outside your site's web tree (this is fully supported by the mobilAP database configuration). Because of its simplicity, it is the recommended option for most mobilAP installations.

Advantages

  • Simplicity - Requires no server setup or software
  • Portable - entire mobilAP data exists as a single file that can be easily backed-up or copied.

Disadvantages

  • Requires permissions changes that may not be possible.
  • Cannot be queried remotely
  • Default installation places it in a folder where anyone can download the data

MySQL

MySQL is a client/server database system. Using MySQL requires an existing server setup with MySQL (although it certainly could be the same server as your web server). Because it exists as a server process, it can be accessed remotely and use a variety of tools to query, update and analyze the data. Access and security can be controlled using standard MySQL administration methods and many hosts and other installations already have existing MySQL servers available. All that is needed is to create a database that has CREATE, SELECT, INSERT, UPDATE and DELETE privileges for the user you are given. Note that setup and configuration of MySQL is beyond the scope of this document.

Advantages

  • Many hosting providers and institutions have MySQL servers available
  • Does not require changing any web server privileges
  • Easier to access remotely

Disadvantages

  • May require setup and configuration
  • Backup/Transfer of data is slightly more complicated
