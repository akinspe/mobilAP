mobilAP Documentation - Overview

MobilAP is a web-based platform for creating interactive conference and classroom experiences. It contains features like question/response, discussion, link sharing and scheduling. The cornerstone of the project is a Mobile Safari based front end that has an optimized experience for users of the iPhone and iPod touch.

Some of the key features include:

  • Web-based setup and administration of content and users
  • Desktop version and Mobile Safari version optimized for iPhone and iPod touch
  • iPhone native version (coming Summer 2010)
  • Schedule of events, including list, day and month views
  • User directory with uploadable photographs
  • Session information including
    • Description and schedule information
    • Presenter/Instructor information
    • Link sharing
    • Interactive Question/response
    • IM style discussion
    • Evaluation
  • Announcements

Schedule

You can use mobilAP with or without a schedule. The schedule allows you to organize sessions by their start and ending time, include room and supplementary information. Administrators can add and edit the schedule. Schedule information is independent from session information, so you can schedule a session as many times as you would like. Scheduling a session multiple times is useful when you want to retain the same links, questions, discussion across multiple times.

Directory

The directory is used to organize the users of your event. Users can browse the directory to get name, contact information and photographs. Administrators can add, edit and remove users.

Sessions

The heart of mobilAP is the session interface. Depending on the context, a session could represent a presentation, a single class meeting or perhaps a series of meetings. Sessions can be administered by the site administrators or by the designed presenters of the session. This allows you to give administrative abilities to users only in the sessions they are presenting. Sessions provide the following services:

  • Link sharing - Users can share web links with others. If desired, the administrators can restrict who can post links
  • Question/Response - Presenters can create multiple choice questions that the users can answer, complete with auto updating charts as people answer.
  • Discussion - IM style discussion where logged in users can participate in a back-channel conversation
  • Evaluation - Users can rate the session based on a site defined series of questions created by the administrators.

Each of these features can be enabled or disabled independently for each session as desired

Announcements

Announcements are useful to convey important information to the users not tied to a particular session. Only administrators can post announcements.
