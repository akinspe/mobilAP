mobilAP Documentation - User/Directory Administration

Managing Users and the Directory

The user directory shows all the users who have been added to the mobilAP user account system. Administrators can manage this list, assign privileges and upload photos as needed. You have several options for creating users:

  • Adding users manually
  • Importing users using a tab-delimited file
  • Allowing users to self-register and create their own account

Note all the steps assume you have logged in as a site administrator. Most of these activities must be done using the desktop interface rather than the mobile interface.

Adding users manually

  1. Click on Directory in the menu
  2. Click the + icon. The detail pane should appear on the right.
  3. Enter the user's first and last names and email (these are required fields)
  4. Optionally enter the user's organization. Because this field is not actually labeled (besides this form), you can use it however you like. In other words, it doesn't necessary have to represent organization.
  5. The admin switch will make or revoke the user's ability to administer the site. Obviously this should only be for user's you completely trust. There is no distinction between administrators. All administrators have the same abilities. This is NOT necessary if you simply want to make a user a presenter
  6. Optionally upload a photo
  7. Click Save

Note: the user's default password will be their email address. For sites where you require passwords for admins or presenters, you should strongly encourage user's to change their password if security is a concern. Use the Setting Administration to change password settings.

Importing Users

If you have many users to add, you can create a simple tab-delimited file and import the users more quickly. You should begin by viewing the example_import.xls file in the import directory of the mobilAP download archive.

Editing the example_import.xls file

  1. Open the spreadsheet file (any spreadsheet program capable of reading MS Excel files and exporting tab-delimited files should work)
  2. The first row contains the field names and order. There are example rows that you can remove
  3. Save/Export the file to tab-delimited format. Please note that comma delimited format is not currently supported. The name of the file is not important, but keep track of where you save the file on your computer.

Alternatively, you can create a tab-delimited file manually using the example_import.tab file as a guide. Once you have this file, you can upload it to your site

Importing the file

  1. Click on Directory in the menu
  2. Click the Import button
  3. Browse and select the file on your computer. This file must be tab-delimited
  4. Click Upload to upload the file. Any errors should be shown
  5. Only users with valid values will be shown, if there seems to be a missing row, check that the values are valid (i.e. First and last name and valid email address)
  6. Verify that the field order is correct. If you need to make corrections you will need to reimport the file
  7. Click Add All to import all the visible users
  8. Optionally you can click Add next to individual users to add a subset of users.

Note: the user's default password will be their email address. For sites where you require passwords for admins or presenters, you should strongly encourage user's to change their password if security is a concern. Use the Setting Administration to change password settings.

Editing a user

  1. Click on Directory in the menu
  2. Click the user you wish to edit in the directory list. The detail panel should appear on the right.
  3. Click the edit button in the detail pane
  4. Update the users's first and last names and email as necessary (these are required fields)
  5. Optionally update the user's organization. Because this field is not actually labeled (besides this form), you can use it however you like. In other words, it doesn't necessary have to represent organization.
  6. The admin switch will make or revoke the user's ability to administer the site. Obviously this should only be for user's you completely trust. There is no distinction between administrators. All administrators have the same abilities. This is NOT necessary if you simply want to make a user a presenter
  7. Optionally upload a photo
  8. Click Reset Password to reset their password to their email address
  9. Click Save to update the data
  10. Click Don't Save to cancel your changes
  11. Click Delete to remove this user completely. This cannot be undone.

Directory Photos

mobilAP supports attaching photos to user's directory entries. This is useful to help people put names with faces and vice versa. Because of the specific nature of photo uploading and processing there are some additional requirements for using the photo upload system in mobilAP:

  • The web server (www user in Mac OS X) must have write access to the mobilAP/data/directory folder
  • File uploads must be enabled in PHP (this is typically not a problem)
  • Either SIPS or GD to process thumbnails (see below)

Uploaded pictures must be JPEG format. After uploading the system will attempt to create a thumbnail it will use one of the following methods:

  • The Scriptable Image Processing System (SIPS), part of Mac OS X
  • The GD Image Library functions (Included on some but not all PHP installations)

If neither of those facilities are available, the operation will fail.

A word about orientation

Some digital cameras support a mode where the orientation of the camera is recorded with the photo and it will rotate the image. Since there are some web browsers that do not recognize this data, mobilAP will attempt to reorient files so their canvas matches their orientation. Support for this feature requires:

  • The Exif functions PHP module (typically enabled on many PHP installations)
  • The GD Image Library functions (Included on some but not all PHP installations)

Presuming those requirements are met, mobilAP should be able to automatically fix orientation problems. Otherwise, you will need to use an image processing program to re-save the files without this data. Using Adobe Photoshop's Save for Web function is a good example.
